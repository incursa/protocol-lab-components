# Go gRPC over HTTP/2 executor

Version 0.3.0 preserves the five unary identities and adds `grpc.h2.server-streaming.echo`, `grpc.h2.client-streaming.echo`, and `grpc.h2.bidi-streaming.echo`. The performance identities run with `grpc-h2-smoke`; diagnostic selection remains available for each exact identity. It opens an authenticated TLS 1.3 connection with mutual ALPN `h2`, creates one HTTP/2 channel, performs an unmeasured validity operation for smoke, then measures one exact RPC on that same channel.

The executor directly observes HTTP/2 status, media type, response trailers, `grpc-status`, the authenticated leaf/SPKI hashes, cardinality, client half-close, stream completion, ordered bidirectional echo, and every deterministic per-message payload/protobuf/gRPC-frame byte scope. Server streaming is one request and 100 responses; client streaming is 100 requests followed by half-close and one final-payload response; bidirectional streaming is one 100:100 ordered RPC, never repeated unary. Gzip is compared after decompression because the public contract deliberately does not make encoder-specific wire bytes comparable. Fixed metadata validates the exact ASCII value and decoded binary bytes. It emits normalized JSON plus aggregate request/response frame streams, the peer certificate, and encoder provenance. The invoking runner remains responsible for preserving stdout and stderr.

Cancellation, deadline-exceeded, trailers-only status, and new-channel identities remain explicit `protocol-lab.unsupported.v1` with exit code 3. Unknown selectors exit 2. The executor never substitutes unary echo or repeated unary for another selector.
